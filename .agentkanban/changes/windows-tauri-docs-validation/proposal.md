# Proposal

## Why

The desktop app introduces a second Windows install path. Users and future contributors need clear
docs, troubleshooting guidance, and validation evidence so the desktop app does not become a hidden
or half-documented branch of the product.

## What changes

- document the Windows desktop app path in README and mdBook
- document desktop architecture and troubleshooting in TECHNICAL notes
- add validation notes for sidecar, docs, tray, and installer behavior

## Non-goals

- large marketing-style desktop-app copy
- Linux/macOS desktop docs before those targets exist
